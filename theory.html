Introduction to Aggregate Functions

Aggregate functions in SQL are used to perform calculations on a set of rows and return a single value. These functions are essential for data analysis and reporting as they allow you to summarize large amounts of data efficiently. Aggregate functions ignore NULL values in their calculations.

COUNT() Function

The COUNT() function returns the number of rows that match a specified condition. It can be used with COUNT(*) to count all rows, or COUNT(column_name) to count non-NULL values in a specific column. 

SELECT COUNT(*) FROM employees;

SUM() Function

The SUM() function calculates the total sum of a numeric column. It is commonly used to find totals such as total sales, total salary, etc. 

SELECT SUM(salary) FROM employees;

AVG() Function

The AVG() function calculates the average value of a numeric column. It returns the mean of all non-NULL values in the specified column. 

SELECT AVG(salary) FROM employees;

MIN() Function

The MIN() function returns the minimum value from a column. It works with numeric, date, and string data types. 

SELECT MIN(salary) FROM employees;

MAX() Function

The MAX() function returns the maximum value from a column. Similar to MIN(), it works with various data types. 

SELECT MAX(salary) FROM employees;

GROUP BY Clause

The GROUP BY clause groups rows that have the same values in specified columns into summary rows. It is used with aggregate functions to group the result-set by one or more columns. 

SELECT department, COUNT(*) 
FROM employees 
GROUP BY department;

HAVING Clause

The HAVING clause is used to filter groups after the GROUP BY clause has been applied. Unlike the WHERE clause, which filters rows before grouping, HAVING filters groups based on aggregate function results. 

SELECT department, AVG(salary) 
FROM employees 
GROUP BY department 
HAVING AVG(salary) > 50000;

Key Differences: WHERE vs HAVING

  • WHERE: Filters rows before grouping and aggregation
  • HAVING: Filters groups after grouping and aggregation
  • WHERE: Cannot use aggregate functions
  • HAVING: Can use aggregate functions
  • WHERE: Applied to individual rows
  • HAVING: Applied to groups of rows

Practical Example

Consider a table employees with columns: id, name, department, salary

-- Find average salary per department with more than 5 employees
SELECT department, AVG(salary) as avg_salary, COUNT(*) as emp_count
FROM employees
GROUP BY department
HAVING COUNT(*) > 5
ORDER BY avg_salary DESC;
